# X Content Strategy Standards

## 三阶段发帖策略

### 阶段一：建立思想影响力（不提项目）
- 发布行业观点，建立个人IP
- 不提 AiProtoMesh，只发思想
- 目标：积累 500+ 精准关注者

### 阶段二：抛出问题，引发讨论
- 开始提出和 AiProtoMesh 相关的核心问题
- 不给答案，让社区讨论
- 目标：形成对话，验证市场认知

### 阶段三：介绍项目
- 此时已有受众，介绍项目有人看
- 用数据和逻辑说话，不用营销语言

---

## 观点矩阵（第一批内容方向）

### 方向一：大模型的未来悖论
```
核心观点：大模型做加法为什么会失去未来

角度：
- 单一模型覆盖所有场景 = 发电厂试图覆盖所有用电需求
- 电网的价值不是发更多电，是让电流动起来
- AI 的未来不是更大的模型，是更聪明的算力调度
```

### 方向二：能源地图重写
```
核心观点：1度电变成 TOKEN，全球能源地图会怎么重写

角度：
- 伊朗 $0.01/KWh 的电 vs 德国 $0.38/KWh 的电
- 能源无法搬运，但算力可以
- 这不是套利，这是全球算力资源的第一次真正流动
```

### 方向三：AI Agent 经济学
```
核心观点：AI Agent 有了预算之后，第一个被颠覆的行业是什么

角度：
- Agent 开始自主雇佣其他 Agent
- AI to AI 不是产品选择，是场景决定的唯一必然
- 当算力成为货币，谁控制管道谁控制未来
```

### 方向四：TOKEN 作为物理锚点
```
核心观点：为什么 1 TOKEN = 1 KW 电力是最聪明的设计

角度：
- 不锚定美元，锚定物理世界
- 电力是唯一真正全球同质化的商品
- 绿色能源 = 绿色 TOKEN = ESG 溢价
```

## 发帖格式规范

### 高互动发帖结构
```
[冲击性开头，1-2行]

[展开论点，3-5行]

[结尾问题或反转，引发评论]
```

### 禁止行为
- ❌ 不发营销语言（"革命性"、"颠覆性"）
- ❌ 不发没有具体数据支撑的大话
- ❌ 不发超过 10 行的长文（除非 Thread）
- ❌ 不在阶段一提项目名称

---

## 半自动发帖 AI 助手构建规范

### 工作流程
```
每日任务：
1. AI 搜集当天 AI / 区块链 / 能源领域热点新闻（3-5条）
2. AI 结合 AiProtoMesh 观点矩阵，生成 3 条发帖草稿
3. 草稿保存到 drafts/[日期].md
4. 人工审核，选择最佳草稿修改后发布
5. 记录发布结果（互动数据）到 analytics/[日期].md
```

### 草稿文件格式
```markdown
# 发帖草稿 [日期]

## 草稿 1
**热点来源：** [新闻标题 + 链接]
**关联观点：** [观点矩阵方向]
**草稿内容：**
[发帖正文]

**预计互动：** 高/中/低
**发布建议时间：** 早9点 / 午12点 / 晚8点

---

## 草稿 2
...
```

### 技术实现
```
工具栈：
- 新闻抓取：RSS Feed + web_search
- 草稿生成：Claude API（结合观点矩阵 prompt）
- 文件存储：本地 markdown 文件
- 发布：人工手动（保留人味）

不需要自动发布，人工审核是刻意保留的环节
```
