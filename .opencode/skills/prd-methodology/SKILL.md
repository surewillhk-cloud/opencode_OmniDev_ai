# PRD Writing Standards

## 强制追问清单（逐一追问，不得合并跳过）

```
1. 目标用户是谁？（具体画像，不接受"所有人"）
2. 核心使用场景？（什么情况下用，在哪里用）
3. 现在怎么解决这个问题？（现有方案的痛点）
4. 成功的标准是什么？（必须可量化）
5. 功能边界是什么？（明确列出不做什么）
6. 优先级排序？（只能做一件事，做哪个）
7. 技术栈是什么？（影响工程师选型）
```

> 如果用户回答模糊，继续追问，直到每条都有明确答案。

## PRD 文档模板

保存路径：`docs/PRD-[功能名]-v[版本号].md`

```markdown
# PRD: [功能名称]
> 版本: v1.0 | 日期: [日期] | 状态: 待评审

## 一句话描述
[用一句话说清这个功能解决什么问题]

## 目标用户
- 用户类型: [具体描述]
- 使用场景: [具体场景]
- 当前痛点: [现有问题]

## 功能需求

### 核心功能（必须做）
- [ ] [功能1] — 验收标准: [具体可测量的标准]
- [ ] [功能2] — 验收标准: [具体可测量的标准]

### 扩展功能（可以做）
- [ ] [功能3] — 验收标准: [标准]

### 明确不做
- [功能X]: [原因]

## 成功指标
| 指标 | 目标值 | 测量方式 |
|------|--------|---------|

## 页面清单
| 页面 | 功能描述 | 优先级 |
|------|---------|--------|

## 技术约束
- 前端: [栈名]
- 后端: [栈名]
- 其他: [如有]

## 迭代记录
| 版本 | 日期 | 修改内容 | 原因 |
|------|------|---------|------|
| v1.0 | [日期] | 初始版本 | — |
```

## 版本规范

| 变更类型 | 版本变化 |
|---------|---------|
| 修复小问题、补充遗漏 | v1.0 → v1.1 |
| 较大结构调整 | v1.1 → v1.2 |
| 需求方向重大变更 | v1.x → v2.0 |
